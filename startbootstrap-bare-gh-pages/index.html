When a site requires both a soft certificate and a smart card authentication to log on, you are initially prompted to select which certificate to use for authentication of a two-way Secure Sockets Layer (SSL) configured environment. In the rich client, when the soft certificate is selected the logon proceeds using the soft certificate, but if the smart card is not inserted in the reader the logon process may still prompt users to insert their smart card. Canceling this prompt causes it to display again, but canceling it a second time allows the logon to proceed successfully using the soft certificate.
